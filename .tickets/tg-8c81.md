# Improve MMU Load/Store Test Coverage

Increase coverage for MMU load/store functions from current 57-62% to 80%+.

Current coverage:
- StoreU8/16/32/64: 57.1%
- LoadU8/16/32/64: 62.5%

Add tests for scenarios the C code handles:
- Store through virtual memory in different privilege modes (M/S/U)
- Load/store with MPRV bit set
- Cross-page boundary access via MMU
- TLB behavior on store operations
- Store to read-only page (should fault)

The goal is to exercise the same code paths C TinyEMU uses during Linux boot.
