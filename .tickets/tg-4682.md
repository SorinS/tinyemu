# Implement full tcp_ctl function

Please see docs/1768939013-tcp-input-full-implementation-plan.md for context.

Implement the complete tcp_ctl() function for handling control sockets.

C Reference: tcp_subr.c:884-915

The function is called when a connection with SS_CTL flag reaches ESTABLISHED state. It:
1. Checks if so_faddr != vhost_addr (line 894)
2. Searches exec_list for matching port/address (lines 896-908)
3. If ex_pty == 3: sets so->s = -1, so->extra = ex_exec, returns 1
4. Otherwise: calls fork_exec() (can stub for emulator)
5. Default case: writes error message to send buffer, returns 0

Return values:
- 1: Connection successful (soisfconnected called)
- 2: CTL_CMD mode (SS_NOFDREF set)
- 0: Error, needs FIN (state goes to FIN_WAIT_1)

Current stub always returns 1, which is incorrect.

## Acceptance Criteria

- Searches exec_list correctly
- Handles ex_pty == 3 case
- Writes error message for unmatched connections
- Returns correct values (0, 1, or 2)
- Matches C behavior at tcp_subr.c:884-915
