# Mbuf needs Dat/Data/Offset fields to match C m_dat/m_data semantics

Parent ticket for Mbuf refactoring. The Go Mbuf struct needs to be refactored to properly model C semantics:

C has:
- m_dat: start of inline buffer (fixed position)
- m_data: pointer to current valid data start (can move within buffer)
- m_size: total buffer size
- m_len: length of valid data

Go should have:
- Dat []byte: backing buffer (equivalent to m_dat)
- Data []byte: subslice of valid data = Dat[Offset:Offset+Len]
- Offset int: where Data starts within Dat
- (Len = len(Data), Size = len(Dat))

This enables:
- Moving data pointer forward (trim from head): Offset += n, Data = Data[n:]
- Moving data pointer backward (prepend): Offset -= n, expand Data
- Correct M_FREEROOM calculation: len(Dat) - len(Data) - Offset
