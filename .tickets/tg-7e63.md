# Poll: check for OOB (out-of-band) data on TCP sockets

The C slirp_select_poll checks the xfds (exceptional fd set) to detect OOB/urgent data:

  if (FD_ISSET(so->s, xfds))
     sorecvoob(so);

The Go processTCPSockets function doesn't check for OOB data availability. This means urgent TCP data won't be handled correctly.

Need to either:
1. Use syscall.Poll with POLLPRI to detect OOB data
2. Or use MSG_OOB with recv() to check/read OOB data

Reference: tinyemu-2019-12-21/slirp/slirp.c:407-408
