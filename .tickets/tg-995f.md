# Add unit tests for mem/IRQSignal - the interrupt signaling mechanism.

## Background
mem/irq.go has **0% test coverage**. This is the mechanism VirtIO uses to signal interrupts to PLIC:

```
VirtIO.ConsumeDesc -> IRQ.Raise() -> PLIC.setIRQ -> CPU.SetMIP
```

If this path has bugs, interrupts will never work.

## Tests Needed
1. NewIRQSignal creation
2. IRQSignal.Raise() calls setIRQ with level=1
3. IRQSignal.Lower() calls setIRQ with level=0
4. IRQSignal.Set() with various levels
5. Nil handler doesn't panic
6. IRQNum() returns correct value

## Acceptance Criteria
- mem/irq.go reaches 100% coverage
- All IRQ signal operations are verified
