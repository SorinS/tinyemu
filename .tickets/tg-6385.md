# Implement TCP_REASS inline fast path

Please see docs/1768939013-tcp-input-full-implementation-plan.md for context.

Implement the inline fast path from the TCP_REASS macro for common case optimization.

C Reference: tcp_input.c:83-99 (TCP_REASS macro)

The macro handles the common case inline:
- Segment is next expected (ti_seq == rcv_nxt)
- Reassembly queue is empty
- Connection is ESTABLISHED

Fast path behavior:
1. Set TF_DELACK (not TF_ACKNOW as current code does)
2. Advance rcv_nxt by ti_len
3. Extract TH_FIN flag
4. Handle so_emu if set
5. sbappend() the data

For out-of-order or complex cases:
1. Call tcp_reass()
2. Set TF_ACKNOW (force immediate ACK)

Current code incorrectly sets TF_ACKNOW for all data.

Depends on: tcp_reass function (tg-82e4)

## Acceptance Criteria

- In-order segments use fast path with TF_DELACK
- Out-of-order segments call tcp_reass with TF_ACKNOW
- TH_PUSH sets TF_ACKNOW (if TCP_ACK_HACK defined, but slirp doesn't use it)
- Matches C behavior at tcp_input.c:83-99
