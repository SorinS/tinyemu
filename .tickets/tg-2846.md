# Add TCP boot test using host forwarding (host-initiated)

Add a TCP test to boot_test.go that verifies host-to-guest TCP using TCPListen/AddHostfwd.

Slirp can listen on a host port and forward incoming connections to a guest address/port. This tests the reverse direction (host initiates connection to guest).

Test approach:
1. Start a simple TCP server inside the guest (using nc -l or a small program)
2. Use slirp.AddHostfwd() or slirp.TCPListen() to forward a host port to the guest
3. From the test, connect to the forwarded host port
4. Verify data can be sent/received through to the guest

This requires the guest to have nc with -l (listen) support, or mounting a simple TCP server program via 9P.

Reference: slirp/slirp.go AddHostfwd, TCPListen
