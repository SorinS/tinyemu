# ip_output ip_off masking operates on single byte instead of 16-bit field

C code does:
  ip->ip_off &= IP_DF;

This masks the full 16-bit ip_off field with IP_DF (0x4000), clearing all bits except the Don't Fragment flag.

Go code does:
  m.Data[6] = byte(m.Data[6] & (IPDF >> 8))

This only operates on byte 6 (high byte of ip_off), masking with 0x40. Byte 7 (low byte) is left unchanged, potentially leaving stale fragment offset bits.

Should be:
  off := binary.BigEndian.Uint16(m.Data[6:8])
  off &= IPDF
  binary.BigEndian.PutUint16(m.Data[6:8], off)
