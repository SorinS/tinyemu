# Comprehensive tcp_input test coverage

Please see docs/1768939013-tcp-input-full-implementation-plan.md for context.

Write comprehensive tests for tcp_input to verify C-matching behavior.

Test categories:

1. Reassembly queue tests:
   - In-order segment delivery
   - Out-of-order segments (2 segments, 3+ segments)
   - Overlapping segments (partial overlap, complete overlap)
   - Duplicate segments
   - Segment trimming (leading, trailing)

2. State machine tests:
   - LISTEN → SYN_RECEIVED transition
   - SYN_SENT → ESTABLISHED transition
   - All FIN-related transitions
   - RST handling in each state

3. Error handling tests:
   - Connection refused → RST sent
   - Host unreachable → ICMP sent
   - Network unreachable → ICMP sent
   - dropwithreset cases
   - dropafterack cases

4. ACK processing tests:
   - Duplicate ACK handling
   - Fast retransmit trigger (3 dup acks)
   - Window updates
   - Congestion window changes

5. Edge cases:
   - Zero window probes
   - Sequence wraparound
   - SYN-in-window attack
   - Small packet escape optimization

Target: 80%+ test coverage for tcp_input.go

Depends on: All implementation tickets

- go test -cover shows 80%+ on tcp_input.go
