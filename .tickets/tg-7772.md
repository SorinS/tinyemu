## Fix Applied

Fixed the SYN_RECEIVED → ESTABLISHED transition bug:

**Problem**: The previous code used `fallthrough` from TCPSSynReceived to the 
ESTABLISHED/FIN_WAIT_1/etc case. This caused the code to enter at the TOP of
the case, hitting the duplicate ACK check: `if seqLEQ(tiAck, tp.SndUna)`.

Since SYN_RECEIVED sets `SndUna = tiAck`, this condition was ALWAYS true,
causing the code to treat the transition as a duplicate ACK and `goto step6`,
skipping all the synrx_to_est processing:
- RTT update (tcp_xmit_timer)
- Retransmit timer handling
- Congestion window adjustment
- Send buffer management

**Fix**: Replaced `fallthrough` with `goto synrxToEst`, matching the C code's
`goto synrx_to_est` at line 892. This jumps directly to the synrx_to_est label,
skipping the duplicate ACK check entirely.

Also fixed indentation issues and removed orphan closing brace at line 868.
