# Preserve IP/TCP headers for RST and ICMP responses

Please see docs/1768939013-tcp-input-full-implementation-plan.md for context.

The current implementation strips headers early and cannot send proper RST packets or ICMP errors. Need to save the original IP header (save_ip) and preserve the ti pointer before stripping headers.

C Reference: tcp_input.c:268-274 saves IP header, tcp_input.c:1268-1278 uses it for RST.

Required:
1. Save full IP header before stripping (like C's save_ip = *ip)
2. Store IP TOS for later use
3. Keep original header data accessible for tcp_respond() and icmp_error()
4. May require architectural change to when/how headers are stripped

## Acceptance Criteria

- save_ip equivalent stored before header stripping
- Original IP TOS preserved
- Header data available for RST/ICMP sending
- Matches C behavior at tcp_input.c:268-274
