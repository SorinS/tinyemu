Please see docs/1768939013-tcp-input-full-implementation-plan.md for context.

Implement the complete TCP reassembly function matching C behavior exactly.

C Reference: tcp_input.c:104-209

The function:
1. Finds insertion point by sequence number (for loop at line 122)
2. Handles overlap with preceding segment - trim or drop (lines 132-153)
3. Handles overlap with succeeding segments - trim or dequeue (lines 160-174)
4. Inserts new segment into queue (line 179)
5. Presents contiguous data to user (present label, lines 181-208)

Key behaviors:
- If ti==NULL, just try to present queued data (goto present)
- Returns TH_FIN if reassembly now includes FIN segment
- In SYN_RECEIVED state, don't present data even if contiguous (line 191-192)
- Handles SS_FCANTSENDMORE by freeing data instead of appending

Depends on: Reassembly queue data structures (tg-28b7)
