# TCP testing: Guest server with TCPListen (host-to-guest)

Implement boot test where host connects to guest TCP server via TCPListen.

## Context
See docs/1769135871-tcp-integration-test-plan.md for full analysis.
See docs/COMMIT_EXPECTATIONS.md for commit standards.

This tests the reverse direction - host initiating connection to guest.

## Implementation
1. Use TCPListen to forward: host:PORT -> guest:8080
2. Boot guest, configure network
3. Guest: nc -l -p 8080 (start listener)
4. Host: net.Dial to localhost:PORT (forwarded to guest)
5. Host sends 'HELLO_FROM_HOST'
6. Guest echoes it back
7. Host receives the echo

## Test Flow
Host (Go net.Dial) --> Slirp TCPListen --> Guest (nc -l server)

## Challenges
- Guest's nc may not have -l option
- nc may exit after first connection
- May need to verify nc capabilities in guest image first

## Acceptance Criteria
- Host connects successfully
- Bidirectional data exchange works
- Test completes without timeout

Reference: slirp/slirp.go TCPListen, machine/boot_test.go
