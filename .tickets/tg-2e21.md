# Compare Go vs C TinyEMU Boot Trace

Run identical boot scenario on both C TinyEMU and Go port with debug tracing enabled. Compare execution to find divergence point.

Steps:
1. Enable debug output in C TinyEMU (compile with debug flags)
2. Add equivalent debug output to Go port
3. Boot same images (bbl64.bin, kernel-riscv64.bin, root-riscv64.bin)
4. Compare instruction traces around the 'Freeing unused kernel memory' point
5. Identify exact instruction/state where Go diverges from C

The goal is transliteration - find where Go doesn't match C behavior, not to 'improve' anything.

Reference: C source is in tinyemu-2019-12-21/
