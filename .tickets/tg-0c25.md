# ip_output does not implement IP fragmentation

The Go ip_output returns -1 and frees the mbuf when a packet exceeds IF_MTU (1500 bytes). The C implementation properly fragments large packets into multiple smaller packets.

This means any IP packet larger than MTU will be silently dropped instead of being transmitted as fragments.

Affected code path: When ipLen > IFMTU, C code (lines 88-164) creates fragment chain and sends each fragment. Go code just calls m.Free() and returns -1.

Reference: tinyemu-2019-12-21/slirp/ip_output.c:88-164
