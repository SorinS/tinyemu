# Add integration test verifying the VirtIO -> PLIC -> CPU interrupt path works correctly.

## Background
Linux boot shows 0 external interrupts (PLIC pending=0x0). We need to verify the interrupt delivery chain works in isolation.

## Test Scenario
1. Create a VirtIO device with PLIC IRQ
2. Submit a request that triggers ConsumeDesc
3. Verify IRQ.Raise() is called
4. Verify PLIC pending bit is set
5. Verify MIP_SEIP/MIP_MEIP is set on CPU
6. Test claim/complete cycle

## Acceptance Criteria
- Integration test passes showing full interrupt path works
- Test exercises: VirtIO -> IRQSignal -> PLIC -> CPU MIP
