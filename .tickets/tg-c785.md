# Add TCP boot test using exec forwarding (guest-initiated)

Add a TCP test to boot_test.go that uses AddExec to configure port forwarding.

When the guest connects to a configured exec address/port, Slirp can route it to a local service. This tests guest-initiated TCP through the exec forwarding path.

Test approach:
1. Start a TCP echo server in the test on the host
2. Use slirp.AddExec() to map a guest address (e.g., 10.0.2.100:8080) to handle connections
3. Have the guest use nc to connect to 10.0.2.100:8080
4. Verify data can be sent/received

Reference: slirp/tcp_subr.go TCPCtl, slirp/slirp.go AddExec
Depends on understanding how ExPty=3 mode works for socket redirection.
