# Add TCP boot test for guest-initiated connections

The current network boot test verifies UDP (DHCP) and ICMP (ping) but doesn't test TCP from within the guest.

TCP from guest requires special setup because connections to vhost (10.0.2.2) go through TCPCtl which returns 'No application configured' unless exec forwarding is set up. This matches C TinyEMU behavior.

To add TCP testing, options are:
1. Use AddExec to configure a port on a guest address that connects to a host service
2. Have the guest connect to a real external address (requires network access)
3. Set up host forwarding (TCPListen) to test host->guest TCP

Reference: slirp/tcp_subr.go TCPCtl function
