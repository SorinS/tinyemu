# IPSlowtimo stub called instead of ipSlowTimo implementation

There are two IP slow timeout functions:

1. ipSlowTimo() in ip_input.go:264 - ACTUALLY IMPLEMENTS fragment timeout logic
2. IPSlowtimo() in device.go:232 - STUB that does nothing

device.go:223 calls s.IPSlowtimo() (the stub), so IP fragment reassembly timeouts never actually happen. Fragments with expired TTL will never be freed, causing a memory leak and potentially incorrect reassembly behavior.

Fix: device.go should call s.ipSlowTimo() instead of s.IPSlowtimo(), and the stub should be removed.

Reference: tinyemu-2019-12-21/slirp/ip_input.c:443-461
