# Implement TCP reassembly queue data structures

Please see docs/1768939013-tcp-input-full-implementation-plan.md for context.

Add data structures needed for the TCP reassembly queue. The C code uses a doubly-linked list of tcpiphdr structures per connection.

C Reference: tcpip.h:60-66 (macros), tcp_var.h (seg_next, seg_prev in tcpcb)

Required changes:
1. Add Next/Prev *TCPIPHdr fields to TCPIPHdr struct for queue linkage
2. Add TiLen field to TCPIPHdr to track segment length (ti_len)
3. Initialize reassembly queue in tcpNewTCPCB() as circular list (seg_next = seg_prev = self)
4. Free reassembly queue segments in TCPClose()
5. Implement helper functions:
   - tcpfragListEmpty(tp) - check if queue is empty
   - tcpfragListFirst(tp) - get first element
   - tcpfragListEnd(ti, tp) - check if at end

## Acceptance Criteria

- TCPIPHdr has Next/Prev/TiLen fields
- TCPCB reassembly queue initialized correctly
- TCPClose() frees queued segments
- Helper functions implemented
- Reference: tcpip.h:60-66
