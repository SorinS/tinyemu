# Create small bare-metal RISC-V programs to test VirtIO in isolation (without Linux).

## Background
Linux boot is complex and hard to debug. Simple bare-metal programs can verify subsystems work before attempting full Linux boot.

## Programs Needed
1. **hello_virtio_console**: Initialize VirtIO console, print 'Hello World'
2. **read_virtio_block**: Initialize VirtIO block, read sector 0, verify contents
3. **virtio_interrupt_test**: Submit VirtIO request, verify interrupt fires

## Implementation
- Write in RISC-V assembly or minimal C
- Run in machine mode (no MMU complexity)
- Use HTIF or custom exit mechanism for pass/fail
- Add to testdata/bare-metal/

## Acceptance Criteria
- Programs run successfully in emulator
- Each program tests one specific subsystem
- Failures are easy to diagnose
