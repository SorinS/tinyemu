# Implement proper tcpDropWithReset with tcp_respond

Please see docs/1768939013-tcp-input-full-implementation-plan.md for context.

The current tcpDropWithReset() silently drops packets without sending RST. Need to implement proper RST sending via tcp_respond().

C Reference: tcp_input.c:1268-1278 (dropwithreset label)

Three cases:
1. TH_ACK set: tcp_respond(tp, ti, m, 0, ti->ti_ack, TH_RST)
2. TH_SYN set: ti->ti_len++, then tcp_respond(tp, ti, m, ti->ti_seq+ti->ti_len, 0, TH_RST|TH_ACK)
3. Neither: tcp_respond(tp, ti, m, ti->ti_seq+ti->ti_len, 0, TH_RST|TH_ACK)

Depends on: Header preservation ticket (tg-7831)
